初めてのプログラミング基礎

プログラミングとは？

プログラミングとは、コンピュータに対して「何をどうしてほしいか」を正確に伝える 手順書（レシピ）を書く作業です。

コンピュータは人間と違って推測できないため、曖昧な指示は理解できません。
例：「いい感じに並べて」→ NG

必要なのは順番・条件・繰り返しを明確に書くことです。

プログラムの3つの基本要素

  • 変数：値を保存しておく「名前付きの箱」
  • 条件分岐：もし○○なら ××する（判断）
  • 繰り返し：同じ処理を何度も行う
age = 20 if age >= 18: print("大人です") for i in range(3): print("こんにちは")

コンピュータができないこと・苦手なこと

  • 曖昧な指示を理解できない
  • 書いた通りにしか動かない（善意の補正がない）
  • 順番が違うと正しく動かない
  • “まあこれでしょ” という推測をしてくれない

エラー（バグ）とは？

プログラムが想定した動きをしない状態をバグと呼びます。 初心者も上級者も必ず何度も経験します。

よくある例：

  • スペルミス（例：pritn → print）
  • カッコの閉じ忘れ
  • 変数名が一致していない
  • 意図しない値が入っている

デバッグの基本

デバッグは “間違い探し” に近い作業です。

初心者が覚えておくべきポイント：

  • エラー文は「ヒント」なので読む
  • どこまで動いたかを確認する
  • 1行ずつ順番に追いかける
  • 小さく試して、少しずつ修正する

アルゴリズムとは？

アルゴリズムとは、問題を解決するための “手順そのもの”です。プログラムはその手順をコンピュータ向けに書いたものです。

例：1〜5の合計を求めたい ↓ 1を足す → 2を足す → 3を足す → 4を足す → 5を足す（これが手順＝アルゴリズム）

データベースで使う演算子と計算

SQL では、データを条件で絞り込んだり、値を計算したりするために 演算子を使います。 これはプログラミング言語とほぼ同じ考え方です。

① 比較演算子（条件を作る）

「どのデータを使うか」を決めるために使います。

= 等しい !=, <> 等しくない > より大きい < より小さい >= 以上 <= 以下

例：

age >= 20 -- 20歳以上 price < 1000 -- 1000円未満

② 論理演算子（条件を組み合わせる）

複数の条件を同時に使いたいときに使います。

AND 両方の条件を満たす OR どちらか一方を満たす NOT 条件を否定する

例：

age >= 20 AND gender = '男' age < 18 OR age >= 65

③ 算術演算子（計算する）

数値の計算も SQL の中で行えます。

+ 足し算 - 引き算 * 掛け算 / 割り算

例：

price * 1.1 -- 税込み価格を計算 salary / 12 -- 月給を計算

④ SQLでは「計算結果」もデータとして扱う

SQL の特徴として、計算結果そのものを表示したり、条件に使ったりできます。

SELECT price * quantity AS total_price FROM orders;

このように SQL は 「データを取り出す」＋「その場で計算する」 ことができる言語です。

SQLを学ぶ前に知っておくと良いこと

  • SQLも「コンピュータへの指示」である
  • 曖昧な書き方をすると意図しない結果になる
  • 文法ミスは必ずエラーになる（コンピュータは融通が効かない）
  • 小さな例で動かして、挙動を確認するのが大切

特に SQL は “順番” よりも “キーワードの並び” が大事です。

初心者へのメッセージ

プログラミングはスポーツと同じで、練習すれば必ず上達します。
最初はわからなくて当然。失敗もエラーも学びの一部です。

このアプリで手を動かしながら、「書く → 動かす → 直す」の循環に慣れていきましょう。
